## 主要決定事項（2026-06-12 改訂版で確定、その後の追補含む）

### 起動・対象者

- 想定対象：自社内・最大 300 人
- 起動：管理者の手動起動のみ（自動トリガー無し、メール起動無し）
- 対象者選定 UI：左側に「全員」チェックボックス 1 個（チェック有=mode=ALL、無=UNREACHABLE_ONLY）

### 認証・利用者

- 認証：Cognito（**Administrator ロールのみ**、一般社員ロールおよびセルフサービス画面は廃止）
- 社員マスタ入力項目：**氏名と電話番号のみ**（その他属性は内部運用用に保持）

### 通話・応答取得

- 安否確認手段：Amazon Connect 自動架電（**音声録音 + Amazon Transcribe**、DTMF 全廃）
- ガイダンス：Polly TTS（言語 ja-JP）、本文は CFn Parameter で変更可能
- ステータス判定：キーワードマッチング（4 区分 SAFE / INJURED / UNAVAILABLE / OTHER）
- 判定優先順位：INJURED > UNAVAILABLE > SAFE > OTHER
- キーワード辞書：管理画面で動的編集、Cycle 起動時にスナップショット辞書バージョンを Cycle に固定

### インバウンド（折り返し電話）

- 受付：本システム代表電話番号への着信を受付
- 身元確認：発信者番号が Employee_Master に完全一致する場合のみ受付（声紋等は使わない）
- 紐付け：直近の RUNNING Cycle、または COMPLETED 30 日以内の Cycle に紐付け、Response を更新
- 受付期間外（30 日超過、未登録番号、Cycle 終了）はガイダンス案内 + 切断 + Inbound_Contact 記録のみ

### 録音・テキスト保管

- 録音：S3 90 日保管後ライフサイクル削除、KMS 暗号化必須
- Transcript：S3 90 日保管後ライフサイクル削除、KMS 暗号化必須
- 録音とテキストの両方を保存（要件 6.3〜6.5、10.4）

### SLA・可用性

- SLA：初回架電 30 分以内、サイクル全体 60 分以内（300 名・並列 10）
- 可用性目標：99.9%（東京リージョン 2 AZ 以上）

### IaC・運用

- IaC：単一 CloudFormation テンプレート、dev/stg/prod を Parameters 切替
- AWS リージョン：東京（ap-northeast-1）固定（**deploy script で `--region` 固定により担保、Phase 15 で実装**）
- ログ：CloudWatch Logs のみ（高度監査ログはスコープ外）
- 電話番号は監査ログでマスキング（先頭 + と末尾 4 桁を残し中間を \*）
- 言語：Python 3.12 / arm64 / boto3 / Hypothesis / mypy(strict) / Ruff / Black（ADR-0001）
- 依存管理：uv

### KMS CMK 段階的実装（ADR-0003）

- Phase 1.5：ルート全権のみ
- Phase 2.x（DynamoDB / S3 着手時）：`kms:ViaService` 制限付き許可を追加
- Phase 6.x（Lambda 着手時）：Lambda / SFN / Transcribe 実行ロール向け Statement を追加
- (4) 明示 Deny は KMS デフォルト動作に委ねる
- `PendingWindowInDays: 7`（最短、Phase 1.5 単独検証時の課金最小化）

## 既知の運用課題（Windows + PowerShell 7 + uv 0.9.30 環境）

1. **AWS CLI for Windows の `file://` UTF-8 デコードエラー**：日本語コメント入り YAML が読めない。`Get-Content -Raw -Encoding UTF8 'path' | %{ $body = $_ }` 経由で渡す
2. **`uv run cfn-lint` が `program not found`**：`.\.venv\Scripts\cfn-lint.exe` 直接呼び出しで回避
3. **Kiro ターミナル UI の表示乱れ**：長いコマンド / 日本語パスでカラム破綻。出力末尾の結果を読み取って判断
4. **`execute_pwsh` の `cwd` 指定が反映されない**：シェルが `backend` ディレクトリに居続けるため、コマンド内のファイルパスを**絶対パス**で指定
5. **`powershell -Command "..."` ラッパーが変数展開を壊す**：外側 pwsh が `$body` を空文字に展開してから子プロセスへ渡すため、ラッパー禁止。直接コマンドを発行
6. **ターミナル出力に過去コマンド残骸混入**：出力冒頭に無関係なコマンドが表示されるが、本体実行は成功。末尾の本来出力（JSON や `EXITCODE: N` 等）で判断
